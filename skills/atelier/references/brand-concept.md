# Brand Concept Workflow

Use this reference for strategy-led identity work before design assets are produced.

## Inputs

Accept product descriptions, founder notes, audience summaries, screenshots, mood references, existing names, competitor lists, or rough aesthetic directions.

## Process

1. Identify the product category, audience, promise, and emotional territory.
2. Extract explicit constraints: industry, tone, taboo aesthetics, required words, language, accessibility, cultural context, and platform needs.
3. Create 3 to 5 visual territories. Each territory must include:
   - concept name
   - strategic rationale
   - mood keywords
   - color direction
   - type direction
   - shape and composition cues
   - image or illustration style
   - motion feel when relevant
   - logo concept seeds
4. Recommend one direction, or present a decision matrix if the user wants options.
5. Produce `BRAND.md` from `templates/BRAND.md` when a durable artifact is requested.

## Quality Criteria

- Use specific, ownable language. Avoid generic phrases such as "modern and clean" unless paired with concrete visual decisions.
- Include anti-patterns so future agents know what not to generate.
- Explain tradeoffs between memorability, flexibility, production complexity, and audience fit.
- Keep the strategy actionable for logo, color, typography, UI, and copy decisions.

## Handoff

When brand direction is approved, route to:

- `logo-system` for marks, wordmarks, lockups, and platform assets.
- `design-refine` if the brand concept should become a complete design system.
- `asset-export` when the user wants a packaged brand kit.
