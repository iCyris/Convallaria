# Convallaria Agent Guide

This file is the source of truth for agents working in this repository. Read it before editing project files, and keep it aligned with `README.md`, `skills/convallaria/SKILL.md`, and `scripts/conva.py`.

## Project Purpose

Convallaria is an agentic design suite for cultivating quiet brand systems from first feeling to final assets. It packages one reusable skill, deterministic helper scripts, Claude Code adapters, Codex plugin metadata, and brand assets into a maintainable local plugin project.

The project should feel like a careful design tool, not a prompt dump. Changes should preserve this standard: specific taste, traceable decisions, deterministic asset operations, and practical handoff files.

## Source Of Truth

- The canonical skill lives in `skills/convallaria/`.
- The main skill entry is `skills/convallaria/SKILL.md`.
- Task classification guidance lives in `skills/convallaria/routing.md`.
- Detailed workflows live as subskills under `skills/convallaria/subskills/`.
- Reusable output structures live beside their owning subskill in `skills/convallaria/subskills/*/templates/`.
- Deterministic asset and validation tools live beside their owning subskill in `skills/convallaria/subskills/*/scripts/`.
- The parent router helper lives in `skills/convallaria/scripts/route_task.py`.
- The Claude Code guide lives at root `CLAUDE.md`.
- Claude Code command adapter sources live in `claude/commands/`.
- Local generated Claude Code files live in `.claude/` and are ignored by git.
- Codex plugin metadata lives in `.codex-plugin/plugin.json`.
- Brand assets live in `assets/brand/`.

Do not duplicate core workflow instructions between platforms. Update the canonical skill or owning subskill first, then sync adapters.

## Protected Dependency

`skills/convallaria/subskills/design-refine/` is a vendored dependency. Do not edit files inside it unless the user explicitly asks to update that subskill.

If the parent skill path changes, moving the vendored folder with the parent directory is acceptable. Editing its contents is not.

## Language Policy

Project files, templates, references, examples, durable deliverables, manifests, and generated handoff artifacts must be authored in English.

If a downstream consuming project needs localization, keep the English source of truth and treat localized files as secondary exports.

## Maintenance Commands

Use the unified maintenance entry point:

```bash
python3 scripts/conva.py doctor
python3 scripts/conva.py sync --no-codex-cachebuster
python3 scripts/conva.py update
python3 scripts/conva.py uninstall
```

Use `--skip-raster` only when an SVG brand asset needs rasterization, the host lacks SVG rendering dependencies, and raster output is not part of the change being verified.

Lower-level scripts exist for implementation:

```bash
python3 scripts/smoke_test.py
python3 scripts/update_convallaria.py
```

Prefer `scripts/conva.py` in documentation and user-facing instructions.

## Required Verification

Before reporting completion after project edits, run:

```bash
python3 scripts/conva.py doctor
```

Also run Python syntax checks after editing scripts:

```bash
python3 -m py_compile scripts/conva.py scripts/smoke_test.py scripts/update_convallaria.py skills/convallaria/scripts/*.py
python3 -m py_compile skills/convallaria/subskills/*/scripts/*.py
```

If verification cannot run, explain why and name the unverified risk.

## Structure Rules

- Keep `.claude/` ignored. It is generated by sync and may contain local user settings.
- Keep root `CLAUDE.md` tracked. It is the source for the Claude Code guide.
- Keep `claude/commands/` tracked. It is the source for Claude Code command adapters.
- Do not commit Python bytecode, caches, local state, or generated runtime folders.
- Keep README examples aligned with real commands that pass `doctor`.
- Keep `README.md` focused on user experience and install/use flow.
- Keep `AGENTS.md` focused on contributor and agent behavior.
- Keep parent skill instructions concise and route detailed procedures to focused subskills.
- Prefer concise one-word names for Convallaria-owned subskills. `design-refine` is the protected vendored exception.

## Design Quality Bar

When changing brand, workflow, or output guidance:

- Tie aesthetic choices to strategy, implementation tokens, logo rules, assets, or explicit open decisions.
- Avoid generic phrases such as "modern and clean" unless paired with concrete visual consequences.
- Preserve craft language, but make outputs useful to designers, engineers, and follow-up agents.
- Ensure multi-file deliverables include `asset-manifest.json` with assumptions, outputs, quality checks, and next actions.

## Codex And Claude Code Integration

Codex reads this project as a plugin through `.codex-plugin/plugin.json`, which exposes `skills/`.

Claude Code reads local adapter files from `.claude/`, generated from root `CLAUDE.md` and `claude/commands/` by:

```bash
python3 scripts/conva.py sync --no-codex-cachebuster
```

After updating plugin metadata, start a new Codex thread after reinstalling or refreshing the plugin so the updated skill metadata is loaded.

Other AI coding tools should start from this file, then read `skills/convallaria/SKILL.md` and the routed subskill under `skills/convallaria/subskills/`. Do not duplicate platform-specific instructions into those tools unless the shared skill contract is insufficient.

## Git Hygiene

The worktree may already contain user changes. Do not revert unrelated edits. If a file has existing changes, inspect it before editing and preserve user intent.

Avoid destructive git commands. Do not use `git reset --hard`, `git checkout --`, or broad deletion commands unless the user explicitly asks for that operation.
